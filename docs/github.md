## Observacoes

A pasta `.git` padrao criada nesta maquina ficou com bloqueio de permissao do Windows. Para manter o versionamento local funcional nesta pasta, foi usado o diretorio de metadados Git alternativo `.gitmeta`.

Ao clonar o repositorio do GitHub em uma nova pasta, o Git volta a funcionar normalmente com a pasta padrao `.git`.
